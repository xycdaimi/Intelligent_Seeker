# 工程说明

## 工程文件结构

1. *.vscode文件夹用于保存vscode工程配置(不使用vscode的话，可以直接删除)*
2. *build文件夹用于保存编译结果*
3. *include文件夹用于保存头文件*
4. *libs文件夹用于保存库文件*
5. *src文件夹用于保存cpp代码文件*
6. *test文件夹用于保存模块测试文件(项目编译时,该文件夹不参与编译)*

## 工程编译

1. *按照工程文件结构将文件放置在对应文件夹内*
2. *通过 bash 直接运行 compile.sh 即可*
3. 可执行文件保存为build/Seeker , 可直接在pc端启动运行
